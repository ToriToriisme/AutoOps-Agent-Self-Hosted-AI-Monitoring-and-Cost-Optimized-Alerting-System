Day 5: Hướng dẫn Cấu hình Grafana Alerting

3.1 Tạo Contact Point (Nơi nhận cảnh báo)

Mục tiêu: Đăng ký một kênh liên lạc (Discord/Webhook) để Grafana có thể gửi tin nhắn đến.

  • Truy cập Grafana, nhìn sang menu bên trái, bấm vào biểu tượng hình cái chuông 🔔 (Alerting), sau đó chọn Contact points.
  • Bấm nút + Add contact point.
  • Đặt tên ở ô Name (Ví dụ: Discord Webhook).
  • Ở mục Integration, chọn Discord hoặc webhook tùy thuộc vào hệ thống bạn muốn nhận.
  • Dán URL của Webhook vào ô yêu cầu.
  • Bấm nút Test góc trên bên phải để gửi thử tin nhắn. Nếu nhận được tin nhắn, bấm Save contact point để lưu lại.

3.2 Thiết lập Notification Policy

Mục tiêu: Cấu hình để mọi cảnh báo hệ thống sinh ra sẽ tự động đi vào kênh Contact Point vừa tạo.

  • Vào menu Alerting -> Notification policies.
  • Tìm phần Root policy - Default for all alerts ở ngay trên cùng.
  • Bấm vào nút Edit (hình cây bút chì) ở góc phải của Root policy.
  • Trong ô Default contact point, chọn tên của Contact Point vừa tạo ở bước 3.1.
  • Cuộn xuống cuối trang và bấm Update default policy.
💡 Lưu ý: Từ bây giờ, bất kỳ cảnh báo nào được kích hoạt đều sẽ gửi thẳng về Discord hoặc Webhook của bạn.

3.3 Tạo Alert Rules (Luật cảnh báo)

Vào Alerting -> Alert rules và bấm nút xanh + New alert rule cho từng luật dưới đây:

🚨 Luật 1: Máy chủ mất kết nối CRITICAL

  • Rule name: Nhập Windows Server Down
  • Bước 1 - Query: Chọn Data source là Prometheus, dán câu query:
    up{job="windows_exporter"} == 0
  • Bước 2 - Behavior:
    - Folder: Nhấn + New folder và nhập Windows Alerts
    - Evaluation group: Nhấn + New evaluation group, nhập Check-1m (khoảng thời gian: 1m).
    - Pending period (For): Nhập 1m.
  • Bước 3 - Details:
    - Labels: Thêm nhãn severity = critical.
    - Summary: Máy chủ Windows đang mất kết nối!.
  • Bấm Save rule and exit.

⚠️ Luật 2: CPU quá tải WARNING

  • Rule name: Nhập Windows CPU High
  • Bước 1 - Query: Dán câu query tính % CPU:
    100 - (avg by (instance) (rate(windows_cpu_time_total{mode="idle"}[2m])) * 100) > 80
  • Bước 2 - Behavior:
    - Folder: Chọn Windows Alerts.
    - Evaluation group: Chọn Check-1m.
    - Pending period (For): Nhập 3m (Chờ 3 phút để tránh báo động giả).
  • Bước 3 - Details:
    - Labels: Thêm nhãn severity = warning.
    - Summary: Máy chủ đang bị quá tải CPU (>80%).
  • Bấm Save rule and exit.

⚠️ Luật 3: Ổ cứng sắp đầy WARNING

  • Rule name: Nhập Windows C: Drive Low Space
  • Bước 1 - Query: Dán câu query tính ổ C:
    100 * (windows_logical_disk_free_bytes{volume="C:"} / windows_logical_disk_size_bytes{volume="C:"}) < 10
  • Bước 2 - Behavior:
    - Folder: Chọn Windows Alerts.
    - Evaluation group: Chọn Check-1m.
    - Pending period (For): Nhập 5m.
  • Bước 3 - Details:
    - Labels: Thêm nhãn severity = warning.
    - Summary: Dung lượng ổ C: sắp đầy (còn dưới 10%).
  • Bấm Save rule and exit.

3.4 Test thử hệ thống TEST

Để đảm bảo mọi thứ hoạt động trơn tru:

  • Vào chỉnh sửa Luật 3 (Ổ cứng), đổi số 10 trong query thành 99.
  • Lưu lại và đợi khoảng 1-2 phút. Bạn sẽ thấy trạng thái của Alert chuyển từ Normal -> Pending -> Firing.
  • Kiểm tra Discord/Webhook xem tin nhắn cảnh báo đã bắn về chưa.
  • Sau khi thành công, nhớ vào đổi lại số 99 thành 10 để hệ thống trở lại bình thường nhé!
